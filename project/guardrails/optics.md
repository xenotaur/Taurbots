# Optics Guardrails

## Research Claims
- Avoid overstating benchmark support, reproducibility, or performance before evidence exists.
- Distinguish local demo functionality from serious ROS/Gazebo/BARN/ArenaBench benchmark readiness.

## User Expectations
- Clearly communicate when a path is lightweight and no-ROS versus when it targets a Linux workstation with heavier robotics dependencies.
- Mark TODOs and unknowns visibly so researchers do not mistake bootstrap intent for implemented capability.

## Attribution and Licensing
- Respect external simulator, benchmark, and dataset licenses.
- Cite upstream benchmark projects when adapters or recipes are added.

## Project Presentation
- Prefer precise language: "intended," "planned," or "candidate" for unevidenced capabilities; "verified" only when backed by evidence.
