# Workstreams

Workstreams group related Taurbots planning and implementation work without replacing the roadmap or individual work items.

Start with:

- `project/roadmap/roadmap.md` for staged sequencing.
- `project/focus/current_focus.md` for the current active priority.
- `project/work_items/` for actionable next steps.

The initial workstreams are:

- `core-research-harness.md`: simulator-independent core, lightweight local loop, PRM-RL first algorithm module, and experiment/result schemas.
- `kubuntu-benchmark-workstation.md`: Linux/NVIDIA workstation readiness, remote workflow, containers, and diagnostics.
- `external-simulator-and-benchmark-adapters.md`: contracts and recipes for BARN, ArenaBench, ROS 2/Gazebo, Webots, MuJoCo, and related validation backends.
